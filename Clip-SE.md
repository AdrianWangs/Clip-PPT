# 研究背景与动机

::left::

### 时代背景：从云存储到大模型

- **数据上云趋势**：海量企业与个人数据存储在云端，数据隐私问题日益凸显。
- **可搜索加密 (SE)**：为解决云端数据“密文可用”问题而生，是隐私计算领域的关键技术。
- **大模型浪潮**：以ChatGPT为代表的大语言模型（LLM）开启了AI新范式。
- **RAG的兴起**：为解决LLM的知识局限和幻觉问题，检索增强生成（RAG）成为主流方案。

*如何在利用大模型能力的同时，保护私有数据的安全，成为新的挑战。*

::right::


### 技术挑战与研究动机

- **RAG的隐私风险**：RAG系统需要将知识库明文存储在云端供检索，存在严重隐私泄露风险。
- **新兴方向：RAG + SE**：将可搜索加密与RAG结合，在保护隐私的前提下增强LLM能力。
- **技术空白**：现有SE方案主要集中于文本检索，无法满足图文并茂的跨模态检索需求。
- **本文动机**：设计一个**高效、安全、支持跨模态**的隐私保护检索方案，为下一代隐私增强RAG提供核心技术支撑。

*本研究旨在填补隐私保护跨模态检索的技术空白，推动可搜索加密在AI时代的应用。*

## 相关工作

<br>

- **SCMR方案**：采用Paillier同态加密技术，允许服务器直接在密文上执行匹配计算。其加密过程引入的随机性会影响最终的匹配结果。
- **PPCMR方案**：使用转置密码进行加密。该方案的密钥管理与用户文件数量相关，并在加密前对齐图文的语义。
- **PITR方案**：要求用户在本地使用HNSW算法来生成一个多层图结构的索引，主要的索引构建计算由客户端完成。

<br>

## 核心组件概览

给定用户图像集 $X = \{x_i\}_{i=1}^n$ 和查询文本 $q$：

::left::

### CLIP特征提取
- **图像特征**：$f_i = \text{CLIP.Extract}(x_i)$
- **文本特征**：$f_q = \text{CLIP.Extract}(q)$
- **语义对齐**：映射到统一512维向量空间

### E2LSH哈希分桶
- **哈希函数**：$H(f) = \lfloor\frac{A \times f + b}{r}\rfloor$
- **参数设置**：$A \in \mathbb{R}^{128 \times 512}$，$B=128$
- **分桶策略**：将128位哈希码分为64段，每段2位
- **快速过滤**：将相似特征映射到同哈希桶，大幅减少候选集

::right::

### 两层加密索引架构
- **第一层索引**：SSE加密的倒排索引（用于快速筛选）
- **第二层索引**：Secure kNN加密的原始特征（用于精确排序）
- **并行处理**：64个SSE索引可同时并行搜索
- **隐私保护**：加密状态下完成所有检索计算

### 工作流程概览
**完整流程：** 本地CLIP特征提取 → LSH哈希分桶 → SSE加密倒排索引 → Secure kNN加密特征 → 云端上传 → 加密检索 → 结果解密

**场景**：用户想从包含`[图像1: 猫]`, `[图像2: 狗]`, `[图像3: 海滩]`的加密数据集中，检索“`一只在海滩上的狗`”。

::left::

### 1. 本地特征提取与加密

- **CLIP特征提取**：
  - 查询文本 → $f_q$
  - 图像1, 2, 3 → $f_1, f_2, f_3$
- **LSH哈希**：
  - $f_q$ → $h_q$ (`01...11`)
  - $f_1, f_2, f_3$ → $h_1, h_2, h_3$
- **两层加密**：
  - **第一层 (SSE)**：基于$h_1, h_2, h_3$构建加密倒排索引 $\tilde{I}$
  - **第二层 (kNN)**：加密$f_1, f_2, f_3$得到 $\tilde{f_1}, \tilde{f_2}, \tilde{f_3}$
- **生成陷阱门**：
  - 基于$h_q$生成SSE陷阱门 $\tilde{T}$
  - 加密$f_q$得到Secure kNN查询 $\tilde{q}$

::right::

### 2. 云端加密检索

- **第一层：快速筛选 (SSE)**
  - 云端使用陷阱门 $\tilde{T}$ 在加密索引 $\tilde{I}$ 中搜索。
  - LSH使语义相似的“狗”和“海滩”的哈希码与查询的哈希码更可能落入同一个桶。
  - **结果**：返回候选集 `{ID_2, ID_3}`，过滤掉不相关的`ID_1`（猫）。

- **第二层：精确排序 (kNN)**
  - 云端仅在候选集 `{ID_2, ID_3}` 中计算精确相似度。
  - 计算 $\text{Sim}(\tilde{f_2}, \tilde{q})$ 和 $\text{Sim}(\tilde{f_3}, \tilde{q})$。
  - **排序**：由于CLIP的语义对齐能力，`Sim(狗, "狗+海滩")` > `Sim(海滩, "狗+海滩")`。
  - **结果**：返回排序后的列表 `[ID_2, ID_3]`。

### 3. 本地解密
- 用户收到加密结果 `[ID_2, ID_3]`，在本地解密，得到最终的图像。

## 局部敏感哈希 (LSH)

::left::

### E2LSH算法原理

**哈希函数：**
$$H(f) = \left\lfloor\frac{A \times f + b}{r}\right\rfloor$$

**参数详细说明：**
- $A \in \mathbb{R}^{B \times D}$：满足标准正态分布的随机投影矩阵
- $b$：$[0,r]$区间内均匀分布的随机偏移向量
- $r$：分桶宽度参数，控制哈希分桶粒度
- $B=128$：生成的哈希码总长度
- $D=512$：输入特征向量维度

::right::

### 哈希码分段与倒排索引构建

**分段策略：**
- 将128位哈希码平均分为64个独立段
- 每段长度：$\lfloor\frac{128}{64}\rfloor = 2$位
- 为每段独立构建一个倒排索引：$I = \{I_j\}_{j=1}^{64}$
- 索引关键字：第j段的哈希值
- 索引值：具有相同哈希值的所有图像ID集合

## 对称可搜索加密 (SSE) 与安全kNN

::left::

### SSE第一层索引：快速筛选

**核心功能：**
- **密钥生成：** $k_{SSE} = \{k_{SSE,j}\}_{j=1}^{64}$（为每个索引生成独立密钥）
- **索引加密：** $\tilde{I}_j^1 = \text{SSE.BuildIndex}(I_j, k_{SSE,j})$
- **陷阱门生成：** $\tilde{T}_j^1 = \text{SSE.GenTrapdoor}(h_{q,j}, k_{SSE,j})$
- **并行搜索：** 云服务器可并行搜索64个SSE索引

**安全特性：**
只泄露搜索模式（相同查询可被识别），但不会泄露查询的具体语义内容

::right::

### Secure kNN第二层索引：精确排序

**密钥组成：**
$$k_{kNN} = \{M_1, M_2, s\}$$  
（两个随机可逆矩阵和一个随机二进制向量）

**加密过程：**
- **特征向量分裂：** 根据向量s将原特征分为两个随机向量
- **特征加密：** $\tilde{f}_i = \{M_1^T f_{i,1}, M_2^T f_{i,2}\}$
- **查询加密：** $\tilde{q} = \{M_1^{-1} q_1, M_2^{-1} q_2\}$

**内积保持特性：**
$$\text{Sim}(\tilde{f}_i, \tilde{q}) = \tilde{f}_i \cdot \tilde{q} = f_i \cdot q$$

<div class="mt-4 p-3 bg-gray-100 rounded text-sm">
<strong>两层协同工作机制：</strong>第一层SSE快速筛选出潜在候选集（通常包含约10%-20%的总数据），第二层Secure kNN在候选集内进行精确相似度计算和排序。这种设计完美平衡了检索效率与准确性。
</div>

## Secure kNN算法流程详解

::left::

### 密钥构成
$$k_{kNN} = \{M_1, M_2, s\}$$
其中：
- $M_1, M_2 \in \mathbb{R}^{d \times d}$ 为随机可逆矩阵
- $s \in \{0,1\}^d$ 为随机二进制向量

1. **向量分裂**：根据s将原特征向量$f_i$分为两个随机向量$f_{i,1}, f_{i,2}$
   - 当$s[j] = 0$时：$f_{i,1}[j] = f_{i,2}[j] = f_i[j]$
   - 当$s[j] = 1$时：随机选择$f_{i,1}[j]$和$f_{i,2}[j]$使$f_{i,1}[j] + f_{i,2}[j] = f_i[j]$
2. **矩阵变换**：$\tilde{f}_i = \{M_1^T f_{i,1}, M_2^T f_{i,2}\}$

::right::

### 查询向量加密步骤
1. **向量分裂**：根据s将查询向量$q$分为两个随机向量$q_1, q_2$
   - 当$s[j] = 1$时：$q_1[j] = q_2[j] = q[j]$
   - 当$s[j] = 0$时：随机选择$q_1[j]$和$q_2[j]$使$q_1[j] + q_2[j] = q[j]$
2. **矩阵变换**：$\tilde{q} = \{M_1^{-1} q_1, M_2^{-1} q_2\}$

<div class="mt-4 p-3 bg-gray-100 rounded text-sm">
<strong>关键设计：</strong>特征向量和查询向量的分裂规则基于相同的二进制向量s，但分裂模式相反，这是内积保持性的关键！
</div>

<!--

首先，Secure kNN的密钥由三个部分组成：两个随机的d×d可逆矩阵M₁和M₂，以及一个随机的d维二进制向量s。

在加密特征向量时，我们分两步进行。第一步是向量分裂，根据随机二进制向量s将原特征向量fᵢ分裂成两个随机向量fᵢ₁和fᵢ₂。分裂规则很巧妙：对于向量中的每个元素位置j，如果s[j]=0，那么两个分裂后的向量在该位置的值都等于原向量的值；如果s[j]=1，那么在该位置随机选择两个数，只要它们的和等于原向量的值即可。第二步是矩阵变换，将这两个分裂后的向量分别用矩阵M₁和M₂的转置进行变换，得到最终的加密特征向量。

## 内积保持性原理

$$
\begin{align}
\text{Sim}(\tilde{f}_i, \tilde{q}) &= \tilde{f}_i \cdot \tilde{q} \\
&= M_1^T f_{i,1} \cdot M_1^{-1} q_1 + M_2^T f_{i,2} \cdot M_2^{-1} q_2 \\
&= f_{i,1}^T M_1 M_1^{-1} q_1 + f_{i,2}^T M_2 M_2^{-1} q_2 \\
&= f_{i,1}^T q_1 + f_{i,2}^T q_2
\end{align}
$$

::left::



**情况一：当$s[j] = 0$时**
- 特征向量分裂：$f_{i,1}[j] = f_{i,2}[j] = f_i[j]$
- 查询向量分裂：$q_1[j] + q_2[j] = q[j]$
- 内积贡献：$f_{i,1}[j]q_1[j] + f_{i,2}[j]q_2[j] = f_i[j](q_1[j] + q_2[j]) = f_i[j]q[j]$

<br>

> $$综上所述：\text{Sim}(\tilde{f}_i, \tilde{q}) = f_{i,1}^T q_1 + f_{i,2}^T q_2 = f_i^T q = f_i \cdot q$$

::right::

**情况二：当$s[j] = 1$时**
- 特征向量分裂：$f_{i,1}[j] + f_{i,2}[j] = f_i[j]$
- 查询向量分裂：$q_1[j] = q_2[j] = q[j]$
- 内积贡献：$f_{i,1}[j]q_1[j] + f_{i,2}[j]q_2[j] = q[j](f_{i,1}[j] + f_{i,2}[j]) = f_i[j]q[j]$

<!--

## 完整的检索流程

::left::

### 索引构建阶段（数据所有者执行）
1. **特征提取**：CLIP提取图像特征向量
2. **LSH处理**：生成128位哈希码并分64段
3. **索引构建**：为每段哈希构建倒排索引
4. **两层加密**：SSE加密倒排索引 + Secure kNN加密原始特征
5. **数据上传**：上传加密图像及索引至云端

<br>

### 查询检索阶段（用户执行）
1. **特征提取**：CLIP提取查询文本特征
2. **陷阱门生成**：生成SSE陷阱门和Secure kNN加密查询
3. **云端检索**：并行搜索SSE索引获候选集，再用Secure kNN精确排序
4. **结果返回**：获取并解密Top-k个最相似图像

  <div>
    <h4 class="font-semibold text-gray-800 mb-1">跨模态能力受限</h4>
    <p class="text-sm">方案强依赖于CLIP模型，目前主要支持图文配对，难以直接扩展至音频、视频等其他模态。</p>

    <p class="text-sm">LSH第一层过滤机制在提高效率的同时，可能误删相关项，导致最终检索精度受到影响。</p>

    <h4 class="font-semibold text-gray-800 mb-1">架构灵活性不足</h4>
